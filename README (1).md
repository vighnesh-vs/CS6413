# Zero-Knowledge Proof for AI Model Validation


The project is split into two phases:

```
Phase 1 — Python                          Phase 2 — C++ (libsnark)
┌──────────┐   ┌──────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│  Train    │──▸│  ONNX    │──▸│Quantize │──▸│ Trusted │──▸│Generate │──▸│ Create  │──▸│ Verify  │
│  Model    │   │  Export  │   │ to int  │   │  Setup  │   │ Witness │   │  Proof  │   │  Proof  │
└──────────┘   └──────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
  PyTorch       .onnx file    circuit.json    PK + VK     Forward pass   Prover(PK,w)  Verifier(VK)
                             round(w×1000)
```

| What | Where it lives |
|---|---|
| Model training + ONNX export | `img_classifier.py` (feature-vighnesh branch) |
| Fixed-point quantization (ONNX → JSON) | `quantize_model.py` |
| R1CS circuit + zkSNARK prove/verify | `src/main.cpp` |

## Prerequisites

**Python (Phase 1):**
- Python 3.8+
- `pip install onnx numpy`
- (Optional) `pip install onnxruntime` for ONNX runtime validation

**C++ (Phase 2):**
- CMake 3.5+
- GCC or Clang with C++11 support
- GMP (`libgmp-dev`)
- Boost (`libboost-all-dev`)
- OpenSSL (`libssl-dev`)
- (Optional) libprocps (`libprocps-dev`) for memory profiling

### macOS (Homebrew)

```bash
brew install cmake gmp boost openssl@3
```

### Ubuntu / Debian

```bash
sudo apt-get install cmake build-essential libgmp-dev libboost-all-dev libssl-dev libprocps-dev
```

## Build & Run

### 1. Clone and initialize submodules

```bash
git clone https://github.com/vighnesh-vs/CS6413.git
cd CS6413
git checkout Khurana7
git submodule update --init --recursive
```

### 2. Run Phase 1 — Quantize model (Python)

Place `zk_mlp.onnx` and `zk_mlp.onnx.data` in the repository root, then:

```bash
python3 quantize_model.py
```

This parses the ONNX graph, quantizes weights with `round(value × 1000)`, and writes `model_data/circuit.json`.

### 3. Build Phase 2 (C++)

**macOS:**

```bash
mkdir -p build && cd build
cmake -DWITH_PROCPS=OFF -DWITH_SUPERCOP=OFF \
      -DOPENSSL_ROOT_DIR="/opt/homebrew/opt/openssl@3" \
      -DCMAKE_POLICY_VERSION_MINIMUM=3.5 \
      -DUSE_ASM=OFF -DCURVE=ALT_BN128 ..
make -j$(sysctl -n hw.ncpu)
```

or USE THIS

```bash
rm -rf build
mkdir build
cd build
cmake -DWITH_PROCPS=OFF -DWITH_SUPERCOP=OFF \
      -DOPENSSL_ROOT_DIR="/opt/homebrew/opt/openssl@3" \
      -DOPENSSL_CRYPTO_LIBRARY="/opt/homebrew/opt/openssl@3/lib/libcrypto.dylib" \
      -DOPENSSL_INCLUDE_DIR="/opt/homebrew/opt/openssl@3/include" \
      -DCMAKE_POLICY_VERSION_MINIMUM=3.5 \
      -DUSE_ASM=OFF -DCURVE=ALT_BN128 \
      -DCMAKE_EXE_LINKER_FLAGS="-L/opt/homebrew/opt/openssl@3/lib" ..
make -j$(sysctl -n hw.ncpu)
```



**Linux:**

```bash
mkdir -p build && cd build
cmake -DWITH_PROCPS=OFF -DCURVE=BN128 ..
make -j$(nproc)
```

### 4. Run Phase 2 — Prove & Verify

From the repository root:

```bash
come to main project directory
cd ..
./build/src/main
```

## How It Works

### Privacy Model

| Role | Knows | Does NOT know |
|---|---|---|
| **Prover** | Input image, model weights, all intermediate activations | — |
| **Verifier** | Verifying key (VK), output logits, proof π | Input image, model weights, intermediates |

The prover generates a succinct proof (~300 bytes) that the classification was computed correctly. The verifier checks this proof in milliseconds using only the public output and the verifying key.

### Fixed-Point Quantization

Neural networks use floating-point arithmetic, but zkSNARKs operate over finite fields (integers mod a large prime). We bridge this gap by multiplying all values by SCALE = 1000 and rounding to integers:

```
float weight  0.1347  →  round(0.1347 × 1000) = 135  (quantized integer)
```

Scale factors accumulate through the network layers:

| Layer | Output Scale | Example Range |
|---|---|---|
| Input (x) | S | [0, 1000] |
| FC1 (z1 = W·x + b) | S² | [-4M, +4M] |
| Square (a1 = z1²) | S⁴ | [0, 19T] |
| FC2 (out = W·a1 + b) | S⁵ | [-18P, +11P] |

All values fit within the BN128 prime field (~2²⁵⁴).
